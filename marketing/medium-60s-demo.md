# 60s Demo — Solo devs / indie makers

**Goal:** longer feature walkthrough for people who already get the category. Pitch DevBoard as their planning + docs workspace.
**Best for:** TikTok longer cuts, Shorts up to 60s, Bluesky video, X video.
**Tone:** narrated voiceover, calm, builder-to-builder.

---

## Hook (first 3s — must earn the watch)

> "I'm a solo dev. I had Obsidian for notes, Miro for diagrams, and a graveyard of `TODO.md` files. So I built one app that does all three — and it's just a folder on my disk."

## Beats

| Time | Screen | Voiceover |
|---|---|---|
| 0–3s | Cold open: split-screen of "before" — Obsidian + Miro + Notion tabs. Smash cut to DevBoard. | "I had three apps for one workflow. So I built DevBoard." |
| 3–8s | Open folder. Workspace loads. Sidebar shows `README.md`, `ROADMAP.md`, `notes/`, `canvases/`. | "Pick a folder. That's your workspace. Markdown notes, canvas pages, all in one tree." |
| 8–15s | Open `ROADMAP.md`. Type `[[v0.4 release]]`. Click it. Show backlinks panel. | "Wikilinks between any note. Backlinks tell you what links back. No plugin to install." |
| 15–25s | Switch to a canvas page called "v0.4 planning". Sticky notes for tasks, shapes grouping epics, arrows showing dependencies. Drop a code block sticky with shell commands. | "Canvas mode is a real infinite whiteboard. Sticky notes, shapes, connectors, code blocks with syntax highlighting." |
| 25–33s | From the sidebar, drag `ROADMAP.md` onto the canvas — it pins as a doc node. Inside `ROADMAP.md`, type `@node:` and pick a sticky from the canvas. Click the mention — jumps to the canvas, focuses the node. | "And the modes talk. Pin a doc onto the canvas. Mention a canvas node from inside a doc. Jump between them with one click." |
| 33–40s | Hit ⌘K. Search "release". Jump to a note. Open quick switcher again, jump to a canvas. | "Quick switcher jumps to any note or canvas in milliseconds." |
| 40–48s | Cut to OS file manager. Show the folder: `.md` files, `canvases/*.json`, `attachments/`. Open one `.md` in VS Code. Run `git status`. | "Everything is plain Markdown and JSON in your folder. Open it in VS Code. Version it with git. Back it up however you back up files." |
| 48–55s | Cut to landing page / download page. Show macOS / Windows / Linux logos. | "Browser version, or desktop app for Mac, Windows, Linux. No account. No telemetry. EU-built." |
| 55–60s | End card: "DevBoard — free, local-first, on Ko-fi" + URL. | "It's free. If it saves you a subscription, the Ko-fi's in the bio. Thanks for watching." |

## Screens to prep

- A real-feeling indie-dev workspace: `README.md`, `ROADMAP.md`, `CHANGELOG.md`, `notes/2026-Q2.md`, `notes/decisions/`, plus 2 canvases (`planning.json`, `architecture.json`).
- The architecture canvas should have rectangles labelled real things ("API", "Worker", "Postgres") with arrows.
- A terminal window with `ls` and `git status` ready in the workspace folder.
- VS Code with the `.md` file open and a syntax-highlighted view.
