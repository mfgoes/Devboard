# 2-minute Deep Dive — Power users / Obsidian crowd

**Goal:** convert the people who already use a notes-graph tool. Show parity on the basics, then the canvas killer feature. End on values (local-first, EU, free).
**Best for:** YouTube Shorts max length, TikTok long-form, embedded on Ko-fi page, pinned reply on Bluesky/X.
**Tone:** narrated, paced, screen-recorded with cursor visible.

---

## Cold open (0–8s)

**Screen:** rapid-cut montage — wikilink resolving, canvas zoom-out, doc pinned to canvas, file explorer showing real `.md` files.

**VO:**
> "Local-first notes app. Markdown and wikilinks, like Obsidian. Real infinite canvas, like Miro or tldraw. Same workspace. One folder. No account. Let me show you."

## Section 1 — The workspace (8–25s)

**Screen:** click "Open folder", pick from disk, watch sidebar populate. Pan over folders: `notes/`, `canvases/`, `attachments/`, `daily/`.

**VO:**
> "DevBoard doesn't have an account. There's no sign-up. You point it at a folder on your disk, and that folder *is* your workspace. Markdown files become notes, JSON files become canvases, everything else stays untouched. You can open the same folder in VS Code, version it with git, sync it with Syncthing, drop it in Dropbox — DevBoard doesn't care, it's just files."

## Section 2 — Notes (25–55s)

**Screen sequence:**
1. Open a note, demo formatting toolbar.
2. Type `[[Project Atlas]]` — autocomplete, click to open.
3. Open backlinks panel — show a list of references, click one to jump.
4. Hit `⌘K`, fuzzy-search to a note.
5. Toggle Focus Mode — the editor goes full-screen, distraction-free.
6. Switch to Stack View — show the vertical scroll of all docs.

**VO:**
> "The editor is clean Markdown with a formatting toolbar if you want it. Wikilinks are first-class — type two brackets, autocomplete, click through. Backlinks panel is built in, no plugin. Quick switcher with command-K jumps to any note or canvas in your workspace. Focus mode for deep work. Stack view if you'd rather scroll through everything as one long document."

## Section 3 — The canvas (55–95s)

**Screen sequence:**
1. Open a canvas page. Empty.
2. Drop a sticky note. Drop a shape. Connect them — show all three connector styles (bezier, straight, orthogonal).
3. Drop a code block sticky, paste in some code, syntax highlights.
4. Drop an image (drag from desktop).
5. Use sections / alignment / snap guides briefly.
6. From the sidebar, drag a `.md` doc onto the canvas — it pins as a live doc node.
7. Switch to that doc. Type `@node:` — picker appears. Pick the sticky. Click the mention — canvas opens, node focused.

**VO:**
> "The canvas is a real infinite whiteboard. Sticky notes, shapes, freeform text, code blocks with syntax highlighting, images, sections, alignment, snap guides. Three connector styles. Undo, redo, duplicate, share-link export. The killer feature for me: you can pin any document onto the canvas as a live node, and from inside any document you can mention any canvas node — `@node` colon, pick one, click it, it jumps you straight there. Your written notes and your visual notes finally know about each other."

## Section 4 — Values (95–115s)

**Screen:** the file manager view of the workspace folder. `git log`. The download page showing macOS / Windows / Linux. The "EU-built" mention on the landing page.

**VO:**
> "DevBoard is local-first. There's no server, no telemetry, no account. Your notes never leave your machine unless you decide to put them somewhere. It's a single-file build that runs offline in your browser, or as a desktop app on Mac, Windows, and Linux. It's built in Europe by one person — me. And it's free. Forever. There's no pro tier, no upsell, no 'free for small workspaces.'"

**VO:**
